# Network Documentation and Assest Inventory

## Objective
Document the lab enviornment by iddentifying host, IP addresses, operation systems, services, and creating a network diagram to establish a baseline inventory

## Tools Used

- Windows 11
- Ubuntu Linux
- Nmap
- Command Prompt
- Terminal
- Draw.io

## Task Performed

- Gathed system information from Windows and Ubuntu
- Documented hostname and IP addresses
- Recorded operation system versions
- Identified services and associated ports
- Created a network diagram
- Built an asset inventory table
- Established baseline documentation for the lab enviornment

## Commands Used

### Windows (cmd)
- ipconfig
- hostname
- systeminfo

### Ubuntu (bash)
- hostname
- ip addr
- lsb_release -a

### Service Enumeration (bash)
- nmap 192.168.1.153
- nmap 192.168.1.78

## What I Learned
- Documentation is essential for managing enviornmetns
- Hostname, IP address, and services should be recorded
- Network diagram provide a visual representation of infrastructure
- Asset inventories help identify systems and their functions
- Oprn ports correspond to services running on devices
- Baseline documentation simplifies troubleshooting and future expansion
- Professional environments rely on accurate documentation
